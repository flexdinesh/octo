# AGENTS.md

## Toolchain

- Node 26.3.1 (`.node-version`). Engine requirement is `>=26`.
- Package manager: `pnpm@11.8.0`. Use `pnpm` exclusively; lockfile is `pnpm-lock.yaml`.
- Monorepo via `pnpm-workspace.yaml`: `apps/*`, `packages/*`. Currently only `apps/api` and `apps/web` exist; `packages/*` is empty.
- Task runner: `turbo`. `turbo.json` defines `build`, `dev`, `typecheck`, `fmt`, `lint`.
- `pnpm-workspace.yaml` sets `verifyDepsBeforeRun: install`, so pnpm may auto-install before command runs.

## Commands

- `pnpm install` — install dependencies.
- `pnpm dev` — run both `api` and `web` dev servers concurrently via turbo.
- `pnpm build` — build all apps via turbo.
- `pnpm typecheck` — typecheck all apps.
- `pnpm lint` — run `oxlint` (Oxc, not ESLint).
- `pnpm fmt` — run `oxfmt` (Oxc, not Prettier).
- `pnpm fmt:check` — check formatting.
- `pnpm --filter <api|web> <script>` — run a script in a single app.

## Lint / format

- Oxc tools: `oxlint` and `oxfmt`. Config in `.oxlintrc.json` and `.oxfmtrc.json`.
- Ignored: `dist`, `coverage`, `.turbo`, `.output`, plus `routeTree.gen.ts` for the formatter.

## App: `apps/api`

- Fastify server. Entry: `src/main.ts`.
- Config from env: `HOST` (default `127.0.0.1`), `PORT` (default `3000`).
- Dev: `node --watch src/main.ts`. Start: `node src/main.ts`.
- Health routes: `/health/live` and `/health/ready`.
- TypeScript extends `tsconfig.base.json` with `NodeNext` resolution, `erasableSyntaxOnly`, `verbatimModuleSyntax`.
- Vitest config at `vitest.config.ts` with `globals: true`. Test script sets `TMPDIR=$PWD/.tmp`. Currently there are no test files, so `pnpm --filter api test` exits with code 1 until tests are added.

## App: `apps/web`

- TanStack Start + Nitro + Vite + React 19 + TailwindCSS v4. See `docs/adr/0001-tanstack-start-nitro-web-app.md`.
- Entry: `src/router.tsx` consuming generated `src/routeTree.gen.ts`; server entry is `server.ts`.
- Dev server runs on `8088` (`vite dev --port 8088`).
- Production build outputs to `apps/web/.output/server/index.mjs`; start with `node .output/server/index.mjs`.
- **Observed runtime behavior**: the production Nitro server currently listens on `http://localhost:3000/` regardless of the `PORT` env var. The integration test in `test/home.test.mjs` fetches `127.0.0.1:3000` and passes. The `Dockerfile` exposes `8088` and sets `PORT=8088`, so the container image will not respond on `8088` unless this is fixed.
- `src/routeTree.gen.ts` is generated by TanStack Router. The formatter ignores it; it is tracked in git.
- TypeScript extends `tsconfig.base.json` with `module: ESNext` and `moduleResolution: Bundler`.

## Testing

- `web`: `node --test test/*.test.mjs`. It builds the prod app, starts it, and performs an HTTP integration test. Do not run it concurrently with another service on port 3000.
- `api`: `vitest run` with globals enabled. Add test files under `apps/api/src/**/*.test.ts` (or equivalent) to make `pnpm --filter api test` pass.

## Docker / deploy

- `compose.dev.yml` builds both apps via their Dockerfiles.
- `apps/api/Dockerfile`: prod install with `--filter api`, starts `pnpm start` on port 3000.
- `apps/web/Dockerfile`: multi-stage build, copies only `.output` to the runner image.

## Conventions

- All packages are `"type": "module"`. Use ESM syntax everywhere (`.ts`, `.tsx`, `.mjs`).
- Strict TypeScript is enforced in `tsconfig.base.json` (`strict`, `noUncheckedIndexedAccess`, `noImplicitOverride`, etc.).
